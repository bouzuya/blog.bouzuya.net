2026-W27 はどうだったか。

2026-06 が終わり、 2026-07 がはじまった。冷房をつけてみたり、その基準に迷ったりしている。

2026-W27 の目標は、詳細な部分で基準が怪しいものの、 [bouzuya/shiori-web] に CLI を追加し export, login のサブコマンドを追加できた。まだ実環境で試していないけど、最小限の実装はできた……つくれたと言っていいと判断した。

なんとなくうまくいかなくてモヤモヤしていた。土曜日にひとつ積読を片付けて、すこしだけ心が穏やかになってはいる。

2026 前半はどうだったか。

もう月ごとのふりかえりも 4 週ごとのふりかえりもしていないのだけど、気まぐれに 2026 の前半をふりかえってみる。

2026 の開発方針は [2026-01-12] に書いた。毎日コミット・毎月 v1 にする、などの方針を書いた。それにそって次のようなことをした。

- 2026-01 tsukota-web v1 : tsukota (Android) の Web 移植 ([2026-01-06], [2026-01-31])
- 2026-02 bbn v5 : blog.bouzuya.net の生成を Rust に移行 ([2026-02-02], [2026-02-25])
- 2026-03 bouzuya-firestore-client v1 : 過去につくった firestore 関連 crate をつかった firestore client ([2026-03-07], [2026-03-28])
- 2026-04 shiori-web v1 : shiori-android の Web 移植・精神的後継 ([2026-04-04], [2026-04-29])
- 2026-05 tsukota-web v2 : 月ごとの集計・ bouzuya-firestore-client v3 の実装と適用 ([2026-05-02], [2026-05-30])
- 2026-06 shiori-web v2 : ダークモード・スマホ対応・シェア設定・ Prev ・ロゴなど ([2026-06-06], [2026-06-15], [2026-06-24])
- 2026-07 shiori-cli : shiori の検索を強化するための CLI ([2026-07-04])

全体的に作り直して整理したり、つくったものを活かせるように……という流れが多い。 Android から Web への移植。 lib crate を集約する lib crate 。その lib を適用した bin 。時間的な制約を意識して、その中で……になっているからかな。新しいものをどんどんつくるとつらくなるので、避けているところがある。

既に何度も停滞している。良くも悪くもコーディングエージェントがあることである程度のものができてしまう。この半年は Claude Code を中心に使ってきた。結局あまり凝った使い方はできていないけど、日常的に素朴には使えている。業務含め、日々「加減」に悩んでいる。

記事を眺めてみると体調が安定していないように見える。あるいは blog にうまく書けない何かがあるときは「体調が悪い」で終えているように思う。昔ほど気楽になんでもは書けていない。

つくったものをふりかえるだけで疲れてしまった。

子どもの小競り合いが激しくなっていて、音量も上がっているように感じる。そういうものにもイライラしてしまう瞬間がある。狭い家に人がたくさん居れば……ね。

もうちょっと掘り下げて、それを踏まえて 2026 後半は……としたかったけど、そこまではできなかった。
